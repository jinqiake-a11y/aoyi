### 3.3 部署方式

#### 方式一：静态服务器部署（推荐）

将 `dist/` 目录部署到任意静态文件服务器：

**Nginx 配置示例**：
```nginx
server {
    listen       80;
    server_name  your-domain.com;

    root   /path/to/skill-tester/dist;
    index  index.html;

    # SPA 路由支持：所有路径指向 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript;
}
```

**Apache (.htaccess)**：
```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>
```

**Caddy**：
```
your-domain.com {
    root * /path/to/skill-tester/dist
    try_files {path} /index.html
    file_server
}
```

#### 方式二：Vite Preview（临时/内网使用）

```bash
# 在服务器上运行预览模式
npm run preview

# 默认访问 http://localhost:4173
# 可通过 --port 和 --host 参数自定义
npx vite preview --port 8080 --host 0.0.0.0
```

#### 方式三：开发服务器（开发/测试用）

```bash
npm run dev
# 默认 http://localhost:3000
```

### 4.2 构建优化建议

当前构建产物约 985KB（JS）+ 40KB（CSS），主要优化方向：

1. **代码分割**：将 SkillDetailDrawer.tsx 按 Tab 拆分为独立 chunk
2. **动态导入**：Three.js 背景组件可改为懒加载
3. **图片压缩**：项目中的截图资源可压缩或迁移至 CDN
4. **移除开发资源**：`e2e/`、`test-results/` 等目录不应部署
